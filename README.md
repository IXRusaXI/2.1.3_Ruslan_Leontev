# 2.1.3_Ruslan_Leontev

Начнем с компонента CounterInfoProps. Как мы видим он в пропсах имеет необязательный value. Соответственно данный компонент будет ререндериться в 2-ух случаях:
1. Изменение наличия value
3. Ререндеринг родительского компонента

Далее App. Мы видим что его состояние зависит от count. Получается что при каждом изменении count через setCount, то-есть по клику на кнопку, будет происходить ререндер.
И так же это будет запускать ререндер всех дочерних компонентов.
